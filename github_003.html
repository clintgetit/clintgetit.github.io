git command

git command

Let's briefly summarize the commands used by git.

git--version

You can check the version of git at the moment.

gitinit

Create a git repository.

gitstatus

View uncommitted changes.

gitadd filename

git add puts the files you have worked on in the local directory into the staging area. In other words, you can declare it as a git-managed file.

git commit -m "Commit message"

Go to staging via git add and commit the files.

In SVN, commit refers to uploading changed files to the server, while commit on git refers to only the changed files (staged files) to the local repository.

In other words, the remote server is not up. To upload to a remote server, you need to run git push again.

gitdiff

Shows the differences between the task tree and the staging area.

Adding the --cached option shows the difference between the staging area and the repository.

Type git diff HEAD to see all of the differences in the working tree, staging area, and repository.

gitmv filename new filename

Change existing file to new file and keep the change history.

gitcheckout - filename

Undo the changes to the previous commit file and revert to the previous commit.

It plays the same role as SVN's revert.

gitbranch

You can see a list of branches in the local repository.

You can use the -r option to view the branches of the remote repository.

gitbranch branch1

Create a new branch called Branch 1 (do not check out).

gitbranch branch2 branch 1

Create a new branch 2 in branch 1.

gitbranch -d branch 1

Delete branch 1.

gotag

You can see a list of tags in your local repository.

You can use the -r option to view the list of tags in the remote repository.

gittag tagname branch 1

Tags in the current state of branch 1 with the tag name.

Regular tags are often used as version names.

gitclone remote store address

Duplicates the contents of the remote storage address and stores it in the current folder.

gitpull

It takes changes from the remote repository and merges them with the contents of the local repository.

gitpush

Uploads content from local storage to remote storage.

gitremote add name store address

Add a new remote repository.

gitremote

Displays the list of remote storage added.

gitremote rmname

Remove the remote repository with that name.

gitcheckout Branch name / tag name

Change the working tree.

gitcheckout -b branch 2 branch 1

In branch 1, create a new branch called branch 2 and change the working tree.

gitrebase branch 1

Apply changes from branch 1 to the current branch.

gitmerge branch 1

The branch of branch 1 is merged with the current branch.
