How can I get simple HTML, JSON?

It is important to handle communication with the server because almost all apps store, retrieve, edit, and delete data on external servers. The following terms are frequently used to communicate with a web server. Below are three definitions for url, session, and task when communicating with a server in Swift. That is, you need to define the address (url) of the web site from which the data will be fetched, the session to the website, and the task to perform certain tasks. The following example shows how to import HTML and JSON data.

Get data fromHTML

We will access the external server and import the HTML. As described above, you need to define url, session, and task to import data from



import Foundation

import PlaygroundSupport

// JSON Get for Playground

PlaygroundPage.current.needsIndefiniteExecution = true

// URL

let urlPath = "https://www.youtube.com/"

let url = NSURL(string: urlPath)

// Session

let session = URLSession.shared

// Task

let task = session.dataTask(with: url! as URL, completionHandler: {

(data, response, error) -> Void in

if error == nil {

let urlContent = NSString(data: data!, encoding: String.Encoding.utf8.rawValue)

print(urlContent ?? "No contents found")

} else {

print("Error occurred")

}

})

// Task Resume

task.resume()



First, url is the address of the site. You must also enter https: // as above. Note that you need to do extra work on info.plist to open the url that just starts with http. We want Swift to recognize the url we see, but Swift will usually recognize it as a String. Therefore, we need to convert this url to NSURL as below.

Next you need to define a session. session is a connection. To get data from a website, you first need to access it. Open the session through URLSession.shared.Now let's take a look at the task. The task requires both the url and session defined above. First, call the session's dataTask method. This allows you to define which url you want to get as an argument value. Then, when the session is connected, have the completionHandler enter what task to perform. The completionHandler closure defines three data, response, and error values ​​that are returned when the session is opened. These values ​​are the values ​​that closure uses as arguments. In particular, data refers to the HTML code in this example, which is the data received when a session is successfully opened.

If there is no error while opening the session, convert the data to NSString first and print it. Do not forget task.resume () at the end.



Get data fromJSON

This time, I will import JSON data from a web page that launches JSON. JSON uses a Chrome extension or JSON Formatter website because it is difficult to understand the structure when opened in a web browser. Now I will import JSON into Swift and print it out.



import Foundation

import PlaygroundSupport

// JSON Get for Playground

PlaygroundPage.current.needsIndefiniteExecution = true

let urlPath = "https://swapi.co/api/people/1/"

let url = NSURL(string: urlPath)

let session = URLSession.shared

let task = session.dataTask(with: url! as URL, completionHandler: {

(data, response, error) -> Void in

do {

let jsonResult =

try JSONSerialization.jsonObject(with: data!,

options: JSONSerialization.ReadingOptions.mutableContainers)

print(jsonResult)

} catch {

print("Error")

}

})

task.resume()



Because this is JSON, not HTML, we converted JSON to an object that Swift can read through JSONSerialization. The method used at this time throws an error and should be handled through do-try-catch.
