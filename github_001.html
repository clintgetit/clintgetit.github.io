1.How can I make a site search on google from a Github site?

GitHub is famous for its free web hosting site. It is important to be exposed to the search site to those who run a personal blog or open a business site. How much exposure is related to profit. If you use the blog service of the portal, you can search it automatically. However, if you use a platform such as github, you have to register to search for each portal directly So, let's look at how to search on google site through Github.

1.Sitemap creation

Create a /sitemap.xml file in the root directory of your blog and copy the contents below. You need to put it in the root directory. If you can not create a sitemap, there are a lot of sites that search on Google and make sitemap.xml free. After updating your blog with git and commit, when you log in with ( blog address/sitemap.xml ), and then you can see the screen to register your sitemap like below.



Information such as lastmod, sitemap.changefreq, sitemap.prioritye of each corresponding article is set in sitemap.xml. It can be additionally set by adding sitemap option at the top of each article as follows. The default setting when there is no setting is defined in

layout: post

title:

date: 2018-10-22 10:46:00

lastmod : 2018-10-22 12:00:00

sitemap :

changefreq : daily

priority : 1.0


2. Create robots.txt
By registering the location of the sitemap.xml file in your robots.txt file, search engine crawlers will help you crawl your homepage. Create a /robots.txt file in the root directory and type.

User-agent: *

Allow: /

Sitemap: http://clintgetit.github.io/sitemap.xml




3. Submit sitemap.xml to Google Webmaster Tools (Search Console)
Access Google Webmaster Tools.
Left Menu - Crawl - Select Sitemaps
In the top right - Select Add / Test SITEMAP.
Enter sitemap.xml in the blank space and test.
If there is no test problem, submit it.

Add the sitemap file to github as below and select submit site map of Google and get approval. Then search on Google will be search.
