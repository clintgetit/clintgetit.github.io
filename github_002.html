Difference between SVN and GIT
SVN vs GIT



What is git?

git is an indispensable version control system for developers.

In the past, companies have used CVS, SVN, and a centralized version control system to do many projects,I use git a lot these days.

If SVN is a centralized version control system, then git is a distributed version control system.

I will look at the overall concept of git and the differences with SVN.



What is a version control system?

A version control system is a system that can record file changes over time and then retrieve the version at a later point in time. It is defined as.

A system that helps you continuously record and recover changes to a file in a database called a repository (repository).

It is said that version control can be done not only for source code but also for graphic files.



Difference between centralized version control system and distributed version control system

The representative of a centralized version control system can be called SVN (Apache Subversion), and the distributed version control system can be called git.

Centralized versioning is done by centralized version control. So the full version of management can only be verified on the server.

Distributed version control systems, on the other hand, mean that full versioning is done on the server, but also on the local store.

In a nutshell, a distributed version control system has the entire contents of a centralized version control system in its entirety in local repositories, and it has the same server.So the commit command is uploaded to the server in SVN, but in git it is the command to save it as local final. (The command to put git on the server needs to run git push.)

Running git push will put the contents of the local repository on the server as a whole.



The distributed version control system basically thinks that version control is local.

Proceeding with version control locally and uploading to remote repository only when the program is perfect and you need to share it with team members or other collaborators.



Advantages of Distributed Version Control System GIT

If remote storage is destroyed or for any reason, it can be restored by simply copying the local storage again,

Version management is possible even when the Internet is not available. It also works locally and is fast



git lient

There is a free git client that makes it easier to use git, a distributed version control system.



Tortoisegit

Tortoise has been a favorite with SVN and has been producing git clients for free.

Usage is similar to Tortoise SVN, so it is easy to integrate with explorer.

The disadvantage is that only for windows.



SourceTree

SourceTree is a GUI based git client created by atlassian famous for JIRA and Confluence.

It is somewhat stable, and since it has all of git's various commands, it is the most stable git client available today.

SourceTree is available for Mac as well as Windows.
