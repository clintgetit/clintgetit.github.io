Various screen sizes?

The screen size and ratio are different for each device such as iPhone, iPad. The iPhone 4 series is 640: 960, the iPhone 5 series is 640: 1136, the iPhone 6 series is 750: 1334, and the iPhone 6 Plus series is 1080: 1920. It is more problematic to have a different ratio than the other. Buttons, labels, tables, and so on, each element's own proportions and their mutual arrangement are tangled or distorted as the aspect ratio changes. Layout problems can occur due to differences in orientations, such as when portrait (portrait) or horizontally (Landscape) the same device (device). Here's how to fix it:



Size Class

Size Class is a new feature since Xcode 8 that allows you to view layouts for each device and orientation directly from the Storyboard, and to apply different layouts for each case. Click the View as part at the bottom of the Storyboard to expand it as shown





Use Aspect Ratio in Storyboard

There is a way to match the size and ratio of the elements in the storyboard to the screen size or other elements by applying Aspect Ratio. In other words, one example is to set the width of a button to 1/2 of the screen and the height to be the same as the width. Before assigning size and scale to each element, group elements through Stackview and apply them.




Use UIScreen.mainScreen() in your code

 

If you want to vary the size of elements such as buttons or labels depending on the aspect ratio, adjusting the Aspect Ratio of Auto Layout is sometimes difficult. In this case, you can use UIScreen.mainScreen to make adjustments directly in the code as below.



UIScreen.mainScreen().bounds.width UIScreen.mainScreen().bounds.height // UIScreen.mainScreen().scale

Each of these properties gets the horizontal and vertical size of the device. The ratio between these two properties is Scale, which is the same value as UIScreen.mainScreen (). Scale. Note, however, that these are point-based rather than pixel-based. For example, for iPhone 6, the width and height would be 375 points x 667 points. This is exactly half the number of 750 pixels by 1334 pixels. That is, point x Scale = pixels.

Defining the layout of elements by screen size and proportions is called a geometry operation. These should be handled by viewWillLayoutSubviews or viewDidLayoutSubviews, not by viewDidLoad. However, keep in mind that these methods are called from time to time. Note that Auto Layout is handled between Will and Did.
